## Основные требования

### 1. Структура данных

- Придумать и использовать **структуру данных** для хранения списка записей истории (массив объектов).
- У каждой записи есть стабильный **`id`** (например `crypto.randomUUID()` или осмысленная альтернатива), чтобы удобно редактировать и удалять.

### 2. Данные — источник истины

- **UI истории** полностью строится из массива в памяти: после любого изменения данных список и счётчики перерисовываются (одна точка входа вида «перечитать данные → отрисовать»).
- Не дублировать «главное состояние» в DOM (например не полагаться на `textContent` списка как на хранилище).

### 3. Добавление записи

- Пользователь может **добавить** запись через форму (модальное окно).
- Опционально: после окончания партии предложить **сохранить результат** с предзаполненными итогом и числом ходов (логика игры уже считает ходы).

### 4. Валидация

- **Пустые значения** не принимать (понятные сообщения об ошибке у полей или под формой).
- **Название записи**: не пустое после `trim`, разумная максимальная длина (согласовать с ментором, напр. 1–80 символов).
- **Итог партии**: только допустимые варианты (X / O / ничья).
- **Число ходов**: только **цифры**, значение в диапазоне **1–9** (как в реальной партии на поле 3×3).

### 5. Вывод списка и фильтр по букве

- Выводить список записей (удобный формат: название, итог, ходы, дата по желанию).
- Строка/набор **букв**: по клику на букву показывать только записи, у которых название начинается с этой буквы (регистр не важен при сравнении).
- У каждой буквы отображать **количество** записей в этой группе (актуально для текущего полного списка; при фильтре по букве список ниже — отфильтрованный).
- Должна быть возможность сбросить фильтр (например кнопка «Все»).

### 6. Изменение записи

- У элемента списка — действие «изменить» (иконка или кнопка).
- Открывается **модальное окно** с теми же полями; сохранение обновляет объект в массиве и перерисовывает UI.

### 7. Удаление одной записи

- Действие «удалить» на элементе; запись **удаляется из массива**, UI обновляется.

### 8. Удаление всех записей

- Отдельная кнопка **«Удалить все»**; желательно подтверждение (`confirm` или вторая модалка).

### 9. Поиск

- Кнопка открывает **модальное окно поиска**.
- В модалке: поле ввода и список **найденных** записей (по подстроке в названии, без учёта регистра).
- В результатах поиска у каждой записи доступны **те же** действия **изменить** и **удалить**, что и в основном списке (общие функции, без копипасты логики).

### 10. Сохранение между перезагрузками

- Использовать **`localStorage`**: при загрузке страницы восстановить массив, при любом изменении — сохранить.
- Обработать битый JSON (например `try/catch` и старт с пустого списка).

## Технический стек

- **Только HTML, CSS, JavaScript** без фреймворков и без сборщика.

## Критерии приёмки (чеклист для ментора)

- [ ] Массив записей — единственный источник истины для блока истории.
- [ ] Добавление / редактирование / удаление / удалить все работают и синхронизированы с `localStorage`.
- [ ] Валидация по правилам выше.
- [ ] Фильтр по первой букве с **подсчётом** записей по буквам.
- [ ] Поиск в модалке + переиспользование редактирования и удаления.
- [ ] Игра «крестики-нолики» по-прежнему работает.
